FileManager Guide

FileManager (file manager) is a module to manage and upload files.

Terms of use
  • This module is XOOPS Cube Legacy only.
  • The module is released under a non-commercial license.
  • The author is not responsible for any damage caused to the user running the module.

Overview

  • Support to upload multiple files at once using the SWF Uploader.
  • Support to create and delete folders.
  • Some functions may not run due to the server environment.

Features

  • XOOPS Cube Legacy full compatible
  • Multiple files upload
  • Image viewer (oliginal size)
  • Thumbnails creation of images and videos
  • Player for music and videos

General Settings

After install FileManager module, configure its default preferences and users permissions.

  • Default upload path
    The default directory to upload is the same of XOOPS Cube Legacy installation: / uploads /
    You can create a new folder under default directory, example: "myfiles".
  • Thumbnail size
    Select the maximum width of thumbnails for file list.
  • Hide System Image
    Do not display images and avatars from system and default "Image Manager".
    By default this option is set to "No" (recommended).
  • Debugging for uploader
    Enable debug screen of SWFUpload to report and help to fix uploader.
    By default this option is set to "No".
  • Upload directory management
    Allow or not webmasters to create and delete folders under directory: / uploads /
    By default this option is set to "No".

Usage

Files and folder names

FileManager only supports alphanumeric characters.
Multi byte file name is not supported.

Files and folders settings

Folders created with FileManager are set with permissions read / write (chmod 0777)
If you plan to give permissions to manage your files with FIleManager but won't delete folders,
it might be wise to set permissions to 0755,0775.

Uploading files can be done under the following permissions : 0755,0775,0777

About FFmpeg

If FFmpeg Library is installed you can crop images and convert video on server side.

  • FFmpeg requires a server-side configuration.
  • FFmpeg binary distribution is not allowed, it's provided separately.
  • FFmpeg operations are dependent of FFmpeg compilation and server environment.
  • Share-hosting might not provide such feature and required resources to convert video.
    For high server load, it's recommended a dedicated server.

Notes

Note that your site can be removed and, or shut down, if you have a limited shared hosting account.

If you create folders and upload files from FileManager, permissions might override FTP settings.

When running a FTP client, the software might recommend that you determine whether you can remove permissions. If you can not remove files or folders from your FTP software, try to do it from the FileManager.
